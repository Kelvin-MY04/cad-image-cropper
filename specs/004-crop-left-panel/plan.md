# Implementation Plan: Image Detection and Cropping — Left Panel Border Crop

**Branch**: `004-crop-left-panel` | **Date**: 2026-03-05 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/004-crop-left-panel/spec.md`

## Summary

Replace the separator-line detection approach (HoughLinesP, single x_coordinate) with a contour-based bold-border rectangle detection approach. `ClassicalBorderDetector` is completely rewritten to detect the drawing panel by its thick rectangular black border frame and return a full bounding box. `DetectionResult` model is updated to carry `crop_region: CropRegion | None` instead of `x_coordinate: int | None`. `SamBorderDetector`, `TwoStageDetector`, and `ImageProcessor` are updated accordingly.

## Technical Context

**Language/Version**: Python 3.12
**Primary Dependencies**: opencv-python-headless ≥ 4.10, numpy ≥ 2.1, Pillow ≥ 11, Typer ≥ 0.12

**Scale/Scope**: Single-image and batch processing; no concurrency changes

## Method Inventory

### `ClassicalBorderDetector` — complete rewrite

**Old methods removed**:
- `_filter_vertical_lines` — finds vertical HoughLines segments
- `_aggregate_x_bins` — bins x positions
- `_compute_y_coverage` — computes y-span coverage
- `_is_in_title_block_zone` — right-zone spatial filter
- `_select_separator_x` — leftmost qualifying x

**Old methods retained (reused)**:
- `_no_detection() → DetectionResult` — DRY helper, unchanged
- `_convert_to_grayscale_array(image_array) → NDArray` — unchanged
- `_apply_binary_threshold(morphed) → NDArray` — unchanged

**New methods**:

| Method | Responsibility | Inputs | Output |
|---|---|---|---|
| `detect_border(metadata, image_array)` | Orchestration: call chain → DetectionResult | ImageMetadata, NDArray | DetectionResult |
| `_isolate_bold_lines(gray) → NDArray` | Morphological open (kernel 4×4) to remove thin lines, then Otsu threshold | gray NDArray | binary NDArray |
| `_find_panel_contours(binary, img_w, img_h) → list[tuple[int,int,int,int]]` | findContours → approxPolyDP → filter qualifying rectangles → list of (x,y,w,h) | binary, width, height | list of tuples |
| `_is_qualifying_rectangle(approx, x, y, w, h, img_w, img_h) → bool` | Checks: 4 vertices, convex, width≥40%, height≥50% | approx poly, bbox, image dims | bool |
| `_select_widest_panel(candidates) → tuple[int,int,int,int] \| None` | Sort by width desc, tiebreak by x asc, return first | list of (x,y,w,h) | tuple or None |
| `_to_crop_region(x, y, w, h) → CropRegion` | Convert boundingRect tuple to CropRegion | x,y,w,h ints | CropRegion |

### `DetectionResult` — model field change

```python
# Before
x_coordinate: int | None

# After
crop_region: CropRegion | None
```

`__post_init__` validation updated:
- `crop_region` MUST be `None` when `method == NONE`
- `crop_region` MUST be non-`None` when `method != NONE`

### `SamBorderDetector` — one method replaced

| Before | After |
|---|---|
| `_extract_x_from_mask(mask) → int \| None` | `_extract_crop_region_from_mask(mask) → CropRegion \| None` |
| Returns `x_coordinate=x` | Returns `crop_region=CropRegion(x_start, x_end, y_start, y_end)` |

Bounding box from mask: `np.where` on row/column axis sums to get min/max y and x.

### `TwoStageDetector` — one-line change

```python
# Before
return result.x_coordinate is not None
# After
return result.crop_region is not None
```

### `ImageProcessor._build_crop_region` — simplification

```python
# Before
return CropRegion(x_start=0, x_end=detection.x_coordinate or metadata.width, y_start=0, y_end=metadata.height)

# After
if detection.crop_region is None:
    raise BorderDetectionError("crop_region is None for non-NONE detection", metadata.file_path)
return detection.crop_region
```

## New Constants

```python
# src/cad_image_cropper/constants.py — add after existing HOUGH_* constants
BORDER_OPEN_KERNEL_SIZE: int = 4          # 4×4 MORPH_RECT destroys ≤3px lines, preserves ≥5px borders
BORDER_MIN_HEIGHT_RATIO: float = 0.50     # panel must span ≥50% of image height
BORDER_MIN_WIDTH_RATIO: float = 0.40      # panel must span ≥40% of image width
BORDER_APPROX_POLY_EPSILON: float = 0.02  # fraction of contour perimeter for approxPolyDP
```

*Note*: `HOUGH_*` and `SEPARATOR_*` and `TITLE_BLOCK_*` constants from feature 003 remain in the file but are no longer imported by `classical_detector.py`. They can be cleaned up in a follow-on task.

## Detection Call Chain

```
detect_border(metadata, image_array)
  │
  ├─ gray = _convert_to_grayscale_array(image_array)
  │
  ├─ binary = _isolate_bold_lines(gray)
  │     └─ morphologyEx(MORPH_OPEN, 4×4) → threshold(OTSU)
  │
  ├─ candidates = _find_panel_contours(binary, width, height)
  │     └─ findContours(RETR_EXTERNAL)
  │          for each contour:
  │            approxPolyDP(0.02 × arcLength)
  │            _is_qualifying_rectangle(approx, bbox, dims)
  │            → append (x, y, w, h)
  │
  ├─ if not candidates → _no_detection()
  │
  ├─ panel = _select_widest_panel(candidates)
  │     └─ sort by w desc, x asc → return first
  │
  └─ return DetectionResult(
         crop_region=_to_crop_region(*panel),
         confidence=None,
         method=DetectionMethod.CLASSICAL
     )
```
